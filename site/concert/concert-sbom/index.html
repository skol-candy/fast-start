    This lab targets Concert v1.0.5.1

    This lab exercise was written and tested against a Concert v1.0.5.1 instance. Please ensure you are running the right version.

    Utilities¶

    syft

    syft is a command line utility for generating a Software Bill of Materials (SBOM) from container images and filesystems.

    curl -sSfL https://raw.githubusercontent.com/anchore/syft/main/install.sh | sudo sh -s -- -b /usr/local/bin

    You can also obtain the syft CLI following the official syft github repository install instructions.

    The best way to install syft for your Mac is using Homebrew

    brew install syft

    You can also obtain the syft CLI following the official syft github repository install instructions.

    You can obtain the syft CLI following the official syft github repository install instructions.

    grype

    grype is a vulnerability scanner for container images and filesystems.

    curl -sSfL https://raw.githubusercontent.com/anchore/grype/main/install.sh | sudo sh -s -- -b /usr/local/bin

    You can also obtain the grype CLI following the official grype github repository install instructions.

    The best way to install grype for your Mac is using Homebrew

    brew install grype

    You can obtain the grype CLI following the official grype github repository install instructions.

    You can obtain the grype CLI following the official grype github repository install instructions.

    trivy

    trivy is a vulnerability scanner for container images and filesystems.

    cat << EOF | sudo tee -a /etc/yum.repos.d/trivy.repo
[trivy]
name=Trivy repository
baseurl=https://aquasecurity.github.io/trivy-repo/rpm/releases/\$basearch/
gpgcheck=1
enabled=1
gpgkey=https://aquasecurity.github.io/trivy-repo/rpm/public.key
EOF
sudo yum -y update
sudo yum -y install trivy

    You can also obtain the trivy CLI following the official trivy github repository install instructions.

    The best way to install trivy for your Mac is using Homebrew

    brew install trivy

    You can obtain the trivy CLI following the official trivy github repository install instructions.

    You can obtain the trivy CLI following the official trivy github repository install instructions.

    cdxgen

    cdxgen is a CLI tool to create a valid and compliant CycloneDX Bill of Materials (BOM) containing an aggregate of all project dependencies in JSON format.

    sudo dnf module enable nodejs:18
sudo dnf install -y npm
curl -s "https://raw.githubusercontent.com/alexdelorenzo/npm-user/main/npm-user.sh" | bash
npm install -g @cyclonedx/cdxgen

    We add the npm-user script so that npm packages can be installed and run as non-root users.

    You can also obtain the cdxgen CLI following the official cdxgen github repository install instructions.

    The best way to install cdxgen for your Mac is using Homebrew

    brew install cdxgen

    You can also obtain the cdxgen CLI following the official cdxgen github repository install instructions.

    You can obtain the cdxgen CLI following the official cdxgen github repository install instructions.

    The best way to install the git utility for your Mac is using Homebrew

    brew install git

    You can also obtain the git CLI following the official Git Documentation.

    You can obtain the git CLI following the official GitHub Documentation.

    application SBOM¶

    To generate an application SBOM, we use the create-application-sbom.sh helper script. The script requires a config file that we can generate dynamically from variables in common_variables.sh. You can view a more comprehensive application config file in the IBM Concert Toolkit sample tempalte.

    using concert-utils 
    source ./common_variables.sh
cat << EOF | envsubst > app-config.yaml
spec_version: "1.0.2"
concert:
  application:
    output_file: "${APP_NAME}-${COMPONENT_NAME}-app-definition-sbom.json"
    app_name: "${APP_NAME}"
    version: "${APP_VERSION}"
    business:
      name: "${BUSINESS_NAME}"
      units:
      - name: "${BUSINESS_UNIT}"
        email: "${BUSINESS_EMAIL}"
        phone: "${BUSINESS_PHONE}"
    properties:
