    Install odf

    The infrastructure nodes were not created during cluster installation so we are going to add these nodes post installation.

    1. List the MachineSets.

    MachineSets are defined in the openshift-machine-api namespace.

    oc -n openshift-machine-api get machineset
    Example output
    NAME                        DESIRED   CURRENT   READY   AVAILABLE   AGE
ocpinstall-btcjq-worker-0   3         3         3       3           109m
    1. Export the worker MachineSet.
    worker_machineset=$(oc -n openshift-machine-api get machineset -o jsonpath='{.items[0].metadata.name}')

    oc -n openshift-machine-api get machineset ${worker_machineset} -o json | jq '. | del(
  .metadata.annotations,
  .metadata.uid,
  .metadata.generation,
  .metadata.resourceVersion,
  .metadata.creationTimestamp,
  .status
)' > infra-machineset.json

    1. Edit infra-machineset.json.

    2. Add labels to spec.metadata.lables:

      • cluster.ocs.openshift.io/openshift-storage: ""
      • node-role.kubernetes.io/infra: ""
    3. Add taint node.ocs.openshift.io/storage: "true" to the spec.
    4. Change all values with postfix -worker-0 to -infra-0 (3 entries).
    5. Change the value of machine.openshift.io/cluster-api-machine-role and machine.openshift.io/cluster-api-machine-type to infra.
    6. Change memoryMiB to 65536.
    7. Change numCPUs to 16.
    8. Change numCoresPerSocket to 2.

    oc apply -f infra-machineset.json

    watch oc get nodes
